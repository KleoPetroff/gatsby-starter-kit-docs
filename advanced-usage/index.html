    Advanced usage

    The ’advanced’ relates to your Git experience. You need some of that to really benefit from Advanced usage, otherwise you are looking for trouble.

    But no worries, you do not need to be a git master, but you should know what git merge is. You should be able to resolve Git conflicts which sometimes come into being after using the mentioned command.

    The Advanced usage installation process is much longer and sweaty than the Default one. But it could be really beneficial afterwards, during the website maintaining.

    What’s the difference?

    With Advanced usage you cannot use gatsby new. The command installs Gatsby website but does not copy git repository to your localhost and there is no Advanced usage without the Git repository.

    With a cloned git repository you will be able to upgrade your localhost code if I refactor codebase in the upstream repo. Sure, sometimes it will not work without resolving conflicts but still.

    With Advanced usage you will be able launch your site as a Website starter and later, when the time comes, upgrade it to Blog by merging the blog branch into your master.

    Git clone

    Instead of gatsby new use the git clone command.

    git clone https://github.com/greglobinski/gatsby-starter-kit.git [NEW_DIRECTORY_FOR_YOUR_SITE]

    Move into the newly created folder.

    cd [NEW_DIRECTORY_FOR_YOUR_SITE]

    List branches of the repo.

    git branch -a

    A ‘develop’ branch

    Now, create a new branch based on the starter/website one. You can call it as you want, I think develop, dev or development are the good ones.

    Make sure that the current branch is the proper one.

    git branch

    You should see that nothing has changed, the starter/website is still the current branch.

      master
* starter/website

    Create a new branch.

    git checkout -b develop

    That’s the output

    Switched to a new branch 'develop'

    List the branches.

    git branch
...
* develop
  master
  starter/website

    The develop branch is the one on which you can apply your changes to the starting content or customize the code of the starter.

    Clear the master branch

    OK, that is the last thing to do for now. You should remember this from the Default usage section.

    The master branch of the Gatsby Starter Kit repository contains code of the starter/blog branch plus some, irrelevant from this point of view, stuff e.g README.md.

    But hey, you installed the Website starter which is lower in the hierarchy of the default starters than the Blog. The Blog is build upon the Website. That means the current master branch is many commits ahead of the newly created develop branch.

    You need to fix this.

    First, you need to delete the current master branch.

    git branch -D master

    And create a brand new one

    git checkout -b master 9eb6079

    Have you noticed the hash at the and of the command 9eb6079? It’s a hash of an initial commit of the Gatsby Starter Kit repo.

    Check it out. Use the git log command to list all of the commits.

    git log --oneline

    Press the space button a couple of times till the moment you see the hash at the beginning of the last row of the listing - 9eb6079 (origin/zero) Add gitignore. Here you are!

    ...
a44eb24 Update content for minimal
5d00dff Copy minimal-content from newbase
421f77b Copy minimal branch code from newbase
9eb6079 (origin/zero) Add gitignore
(END)

    Now you current branch should be the newly cleared master branch.

    git branch
...
  develop
* master
  starter/website

    And merge the develop branch into it.

    git merge --no-ff develop
